# Bio-Framework v1.0.0

## 从原始数据到 SCI 论文初稿，AI 驱动的一站式生物信息学分析框架

Bio-Framework 是一款专为生物信息学研究人员设计的 Claude Code 技能插件。它将数十年积累的生物信息学最佳实践编码为结构化工作流，让 AI 能够像资深分析师一样系统性地完成从数据质控到论文初稿的全流程工作，同时将数据完全保留在你的本地环境中。

## 一、你面临的问题

**生物信息学分析耗时数周，瓶颈却往往不在计算**

真正的时间成本在于：不知道选哪条分析路径、不确定质控阈值是否合理、图表投稿时被审稿人指出不符合期刊格式要求、分析完成后论文写作又是一个全新的挑战。

Bio-Framework 针对这六个核心痛点提供系统性解决方案：

**1. 分析耗时数周**
无结构的探索式分析让时间无从把控。Bio-Framework 内置 6 种组学标准流水线，每一步都有明确的执行规范和质量门控，将分析过程系统化。

**2. 新手不知从何下手**
没有经验积累的研究者面对 Seurat、Scanpy、DESeq2 等工具组合无从选择。框架内置行业标准分析流水线和参数最佳实践，内置 COMMON_ERRORS 知识库避免常见陷阱。

**3. 图表不符合 SCI 投稿标准**
生成图表容易，生成符合 Nature / Cell Reports 格式要求的图表需要经验积累。Bio-Framework 内置 SCI 设计标准（sci_design_standards.yml）和视觉审查门控，每一张图表都经过 AI 视觉检查和用户确认两道关卡。

**4. 分析结果缺乏深度审计**
分析完成后你是否真的确信结果是可靠的？Step 1.5 五维度数据质量深度审计（数据完整性、统计显著性、生物学合理性、方法论合规性、可重复性）在分析推进前强制执行，ULTRATHINK 模式确保每个维度都经过深入推理。

**5. 论文写作是另一个耗时工程**
分析完成只是完成了一半工作。Bio-Framework Step 6 从 Methods、Results 到 Discussion 自动生成 SCI 稿件初稿，经过 6 步精细化后完成期刊合规验证，输出结构化的 SCI 初稿。

**6. 数据安全顾虑**
患者数据、未发表的科研数据不能上传第三方云平台。Bio-Framework 完全运行在 Claude Code 本地环境，数据永远不离开你的计算机。需要注意的是，Claude Code 本身作为 AI 服务会与 Anthropic 服务器通信（发送代码和输出摘要供模型处理），但原始数据矩阵不在传输范围内，详情请参阅 Anthropic 的数据处理政策。

## 二、核心功能展示

### 场景 1：scRNA-seq 全流程分析

用一条命令启动分析，框架自动加载 scRNA-seq 标准流水线，按阶段推进：

```
/bio start --topic topic.yml
```

**框架自动执行的工作：**

- Step 0：解析 topic.yml，识别数据类型、研究问题、样本信息，生成执行计划
- Phase 1：数据加载与质控（线粒体比例、基因数分布、doublet 检测）
- Phase 2：标准化、降维（PCA → UMAP）、聚类
- Phase 3：细胞类型注释（基于 marker 基因库 + 文献核查）
- Phase 4：差异表达分析、通路富集
- Step 1.5（质控门）：五维度深度审计，AI HARD_STOP 机制确保结果可靠后方可继续
- Step 2：反思与探索，验证分析方向的科学合理性
- Step 3：研究摘要生成，完整度验证确保必要图表一一核查
- Step 4：出版质量图表生成，AI 视觉检查 + 用户视觉审查门
- Step 5：综合分析报告
- Step 6：SCI 稿件初稿

**典型输出：**
UMAP 细胞类型大图（强制验证）、Violin 图、Dotplot、差异表达火山图、通路气泡图、Methods 章节、Results 章节初稿。

---

### 场景 2：出版质量图表自动生成

Bio-Framework 不只是"生成图表代码"，它构建了完整的图表质量闭环：

**Step 4 工作流：**

1. **4a 图表规划**：读取 figure_types.yml 知识库，确定每种组学类型的强制图表清单（mandatory: true），防止遗漏关键结果图
2. **4b 代码生成**：生成符合 SCI 设计标准的图表代码（分辨率 300 DPI、矢量导出、正确字体大小）
3. **4c 代码执行**：运行代码生成实际图像文件
4. **4d AI 视觉检查**：框架使用 Read 工具实际打开并检查每一张图表，验证内容正确性
5. **4e 图表索引归档**：生成 table_index.yml 清单，记录所有输出
6. **4f 用户视觉审查门**：展示图表给用户确认，用户批准后方进入下一步

期刊合规检查覆盖 Nature 的图宽要求（89mm / 183mm）、Cell Reports 的色彩规范、Nature Methods 的方法图标准等。

---

### 场景 3：SCI 论文稿件自动生成

Step 6 是 Bio-Framework 核心功能之一：

**6 步精细化流程：**

- **6a**：初始稿件起草（Methods + Results + Discussion + Abstract + Title）
- **6b**：科学准确性审核（与分析数据交叉核实）
- **6c**：叙事逻辑优化（论点连贯性、结论支撑强度）
- **6d**：语言精炼（学术写作风格、被动语态规范、动词时态）
- **6e**：期刊规范对齐（字数限制、图表数量、参考文献格式）
- **6f**：最终投稿准备检查（submission_readiness.md 输出）

目标期刊可在 topic.yml 中指定，框架自动调取对应期刊的 data_requirements 和格式要求。

---

### 场景 4：多组学类型无缝切换

Bio-Framework 支持 6 种组学类型的标准流水线，每种都有独立的分析流程和强制图表清单（另有 ATAC-seq 等组学的知识库覆盖）：

| 组学类型 | 核心分析流程 | 强制标准图 |
|---------|------------|----------|
| scRNA-seq | QC → 降维 → 聚类 → 细胞注释 → DEG | UMAP 细胞类型图（mandatory） |
| 空间转录组 | 空间 QC → 空间降维 → 空间聚类 → 空间基因表达 | 空间散点图（mandatory） |
| Bulk RNA-seq | QC → 标准化 → DEG → 富集分析 | 火山图、热图 |
| 蛋白质组 | 标准化 → 差异蛋白 → 功能富集 | Volcano Plot |
| 代谢组 | QC → 统计分析 → 代谢物注释 | PCA 得分图 |
| 脂质组 | 脂质 QC → 差异分析 → 脂质分类 | 脂质分类图 |
| ATAC-seq | Peak calling → 差异峰 → motif 富集 | 可及性图谱 |

当检测到分析类型时，框架自动探测正确的流水线，mandatory 图表清单在 Step 3 完整度验证中强制执行 HARD_STOP，确保关键图表不被遗漏。

## 四、常见疑问：Claude + Ultrathink 已经很聪明了，为什么还要买 Bio-Framework？

**一句话回答**：Claude 是发动机，Bio-Framework 是整辆车。你需要的不只是马力，还有方向盘、导航、安全气囊和行车记录仪。

| 维度 | Claude + Ultrathink 单独使用 | + Bio-Framework |
|------|---------------------------|-----------------|
| 跨会话记忆 | 每次从零开始，参数和进度全部丢失 | `/bio continue` 精确恢复到中断的子步骤 |
| 分析标准化 | 每次可能推荐不同的工具和参数 | 6 种标准流水线，50+ 领域共识参数 |
| 质量门控 | 无强制检查点，可能带着错误跑完全程 | 五维度强制审计，发现严重问题自动停下等你决策 |
| 错误处理 | 每次从头推理，不记得上次的修复 | 16 种已知模式 + 跨项目经验自动积累 |
| SCI 图表 | 不自动验证期刊格式要求 | SCI 标准 + AI 视觉检查 + 用户审查门 |
| 可重复性 | 不自动记录种子、版本、参数 | 全程自动记录，Methods 一键生成 |
| 稿件生成 | 可写段落，无系统化 SCI 稿件流程 | 21 步端到端稿件（撰写+精炼+验证） |
| 思维调度 | 全程同一深度，要么浪费要么不足 | 4 级自动调度，关键判断强制深度思考 |

**类比**：电子病历系统不比医生聪明，但你还是会用它——因为它让工作更规范、更高效、更可追溯。Bio-Framework 对 Claude 的作用完全相同。

**你购买的不是 Claude 的智力，而是一套经过反复验证和优化的生物信息学专业系统。**

## 三点五、可培养的 AI 科研助手

Bio-Framework 不是一个用完即丢的工具。它设计了完整的个性化和知识积累机制，使用越久，越能理解你的分析风格和研究需求。

### 1. 参数偏好记忆

通过三层配置系统，你的分析偏好可以持久保存：

```
# 将你偏好的聚类分辨率保存到全局设置
/bio config set clustering.resolution 0.6 --global

# 此后所有项目自动使用 0.6，无需每次指定
# 某个特殊项目需要 1.2 时，在项目层覆盖即可
```

偏好保存在 `~/.claude/bio-framework/rules/user_rules.yml`，涵盖质控阈值、可视化参数、差异表达标准等 12 个参数域，跨所有项目生效。

### 2. 经验库自动积累

AI 在分析过程中遇到并解决的问题会自动沉淀为经验：

```
问题: Seurat FindMarkers() 报错 "not enough cells"
解决: 降低 min.pct 阈值至 0.1，检查 cluster 8 细胞数量
         ↓ 自动保存到 learned_solutions.yml
下次遇到同类问题 → 直接调用已有方案，无需重复排查
```

经验库位于 `~/.claude/bio-framework/user_knowledge/learned_solutions.yml`，跨项目共享，保留解决方案、适用条件和复用次数。

### 3. 自定义知识扩展

你可以向用户层添加自定义知识文件，框架会与内置知识**合并**（union 策略），而非替换：

- 添加你的实验室常用 marker 基因列表
- 添加特定疾病领域的分析注意事项
- 添加你的 QC 经验规则

这些知识文件在框架升级时不会被覆盖。

### 4. 分析上下文持久化

每个项目的分析进度、关键决策、参数选择、异常发现全部持久保存：

| 持久化内容 | 文件 | 用途 |
|-----------|------|------|
| 工作流状态 | `workflow_state.yml` | 中断后精确恢复 |
| 阶段摘要 | `phase_summaries.yml` | 记录每阶段的关键决策和参数 |
| 分析发现 | `findings.yml` | 追踪异常和洞察 |
| 上下文快照 | `/bio prime` | 新会话快速恢复语境 |

### 实际效果

第 1 个项目：你需要告诉 AI 你的 QC 标准、偏好的分辨率、图表风格。
第 3 个项目：AI 已经知道你倾向于更严格的 QC 阈值、更细的聚类、特定的配色方案。
第 10 个项目：AI 积累了你的领域经验、常见问题解决方案和分析习惯，成为了解你研究风格的科研助手。

## 四、适用人群

**生物医学科研人员（博士生、博后、PI）**
有分析经验，需要提升效率和输出质量。Bio-Framework 让你专注于科学问题，而不是流程管理。Step 6 稿件生成直接节省论文写作时间。

**生物信息学初学者（实验背景转生信）**
没有系统训练，不确定标准流程是什么。框架内置最佳实践，COMMON_ERRORS 知识库防止常见错误，可以作为有效的学习工具。

**临床研究团队**
患者数据不能离开本地服务器。Bio-Framework 完全本地运行，数据安全有保障，同时提供 Bulk RNA-seq 和蛋白质组等临床研究常用分析流水线。

**多组学研究实验室**
同时处理多种组学数据，需要统一的分析框架和报告标准。Bio-Framework 的 6 种组学标准流水线 + 统一报告格式满足这一需求。

**以 SCI 发表为目标的研究者**
需要高效产出符合期刊标准的图表和稿件。Bio-Framework 将期刊要求编码为自动验证规则，从分析到投稿实现系统化。

以下是将 Bio-Framework 成功推向市场需要额外准备的材料和策略，供参考：

### 内容素材

**1. 演示视频（最高优先级）**
录制 3-5 分钟的产品演示视频，完整展示 scRNA-seq 全流程：从 `/bio start` 到最终 UMAP 图和稿件初稿。无声音的录屏也有效，但配旁白效果更好。建议工具：OBS Studio + 后期字幕。

**2. 案例研究**
准备 2-3 个真实项目的案例分析（可匿名化数据）：展示输入数据规模、分析耗时对比（手动 vs Bio-Framework）、最终输出物（图表 + 稿件片段）。案例是最有说服力的销售内容。

**3. 对比基准测试**
量化数据：Bio-Framework 完成标准 scRNA-seq 分析的时间，vs 有经验的分析师手动完成的时间。即使是粗略的数字（如"分析配置从 2 小时缩短到 30 分钟"）也能显著提升转化率。

### 产品策略

**4. 免费试用版 / 精简版**
提供功能受限的免费版本（例如：仅 scRNA-seq 流水线，无 Step 6 稿件功能），让用户在付费前体验产品价值。降低决策门槛是 SaaS 工具的标准策略。

格式：博客文章 + 配套视频（可先做文字版，逐步补视频）。

**6. 产品官网 Landing Page**

## 八、快速开始

### 安装

Bio-Framework 作为 Claude Code 技能插件安装，需要 Claude Code 运行环境。

```bash
# 克隆仓库
git clone https://github.com/bio-framework/claude-bio.git

# 进入安装目录
cd claude-bio

# 运行安装脚本
./install.sh
```

安装完成后，Claude Code 将自动加载 Bio-Framework 技能（SKILL.md，约 550 行核心指令）。

### 启动你的第一个分析项目

**第一步：创建项目配置文件**

```yaml
# topic.yml
project:
  name: "my_scrna_project"
  language: "zh"  # 或 "en" / "ja"

data:
  type: "scRNA-seq"
  path: "./data/raw/"
  samples: ["sample1", "sample2"]

research:
  question: "识别肿瘤微环境中的免疫细胞亚群"
  target_journal: "nature_methods"
```

**第二步：启动分析**

```
/bio start --topic topic.yml
```

**第三步：按框架引导逐步推进**

框架将自动加载 scRNA-seq 流水线，在每个关键节点提示你确认，在 Step 1.5 执行深度质量审计，在 Step 4 展示图表供你视觉审查，最终在 Step 6 生成稿件初稿。

**查看分析状态：**

```
/bio status
```

**从中断点恢复：**

```
/bio continue
```

**需要帮助：**

```
/bio help
```
