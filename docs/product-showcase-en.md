# Bio-Framework v1.0.0

## From Raw Data to SCI Manuscript Draft — AI-Powered End-to-End Bioinformatics Analysis Framework

You define your research question; Bio-Framework handles the rest.

## The Problems We Solve

### 1. Bioinformatics analysis takes weeks or months

Standard single-cell RNA-seq analysis — QC, normalization, clustering, annotation, differential expression, visualization — involves dozens of interdependent steps, parameter choices, and debugging cycles. Bio-Framework encodes the full standard pipeline and executes it as a structured, automated workflow. What previously took a seasoned analyst days to set up runs end-to-end with a single command.

### 2. Beginners do not know where to start

Moving from wet lab to computational analysis is one of the steepest learning curves in modern biology. Bio-Framework provides built-in standard analysis pipelines for 7+ omics types, each encoding community best practices (10x Genomics / Seurat / Scanpy standards for scRNA-seq; DESeq2 / edgeR standards for bulk RNA-seq; etc.). No prior bioinformatics experience is required to run a complete, publication-standard analysis.

### 3. Figures do not meet SCI submission standards

Journals reject figures with incorrect resolution, font sizes, color accessibility issues, or inconsistent labeling. Bio-Framework's Step 4 generates publication-quality figures against built-in SCI design standards, performs AI visual inspection (reading each PNG via the Read tool), and requires explicit user visual review before proceeding — closing the loop between code output and human-verified quality.

### 4. Analysis results lack rigorous auditing before interpretation

It is easy to proceed to biological interpretation without noticing sample swaps, batch effects that were not corrected, or cross-phase inconsistencies. Bio-Framework's Step 1.5 — Data Quality Deep Review — performs a five-dimensional audit across all completed analysis phases before any interpretation begins:

- Dimension 1: Sample Identity Verification
- Dimension 2: Batch Effect Assessment
- Dimension 3: Biological Plausibility
- Dimension 4: Outlier Sample Detection
- Dimension 5: Cross-Phase Consistency

Critical issues trigger a hard stop requiring user decision; warnings proceed with a logged flag; clean results auto-advance.

### 5. Writing the paper is another project entirely

After analysis is complete, translating results into a structured SCI manuscript typically takes weeks. Bio-Framework's Step 6 auto-drafts a full manuscript — Title, Abstract, Introduction, Methods, Results, Discussion, Figure Legends, References — with 6 sequential polishing rounds (drafting then refinement for each section), followed by 5-phase submission readiness validation. The output is a structured first draft ready for researcher review, not a rough skeleton.

### 6. Data security and regulatory concerns

Cloud-based AI analysis tools require uploading raw data to third-party servers, creating compliance risks for clinical datasets, patient-derived samples, or proprietary data. Bio-Framework runs entirely within Claude Code on your local machine. Raw data files (FASTQ, BAM, count matrices) never leave your environment.

**Important note**: Bio-Framework runs inside Claude Code, which communicates with Anthropic's API. Code snippets, result summaries, and conversational context are sent to Anthropic as part of normal Claude Code operation. Raw data files are not transmitted. If your institution prohibits any analysis-related text from leaving on-premises servers, consult your data security officer before use.

## Core Feature Showcase

### Scenario 1: scRNA-seq Full Workflow

```
/bio start
```

Bio-Framework reads your `TOPIC.yml` (research question + data paths), dynamically plans analysis phases, and executes the full single-cell pipeline:

- QC: cell filtering by nFeature, nCount, percent.mt; doublet removal
- Normalization: SCTransform or NormalizeData/LogNormalize
- Dimensionality reduction: PCA → UMAP/tSNE
- Clustering: graph-based (Seurat FindClusters / Scanpy leiden)
- Cell type annotation: marker-gene-driven, with literature cross-reference
- Differential expression: Wilcoxon / DESeq2 pseudobulk
- Mandatory figures enforced: UMAP colored by cluster, UMAP colored by cell type, violin plots, heatmaps

All standard pipeline steps are tracked for compliance. If a step is skipped without justification, pipeline compliance flags it.

Interrupt at any point; `/bio continue` resumes from exact checkpoint.

---

### Scenario 2: Publication-Quality Figures with Visual Review Gate

After analysis completes, Step 4 generates figures driven by the `figure_plan.yml` produced in Step 3:

1. **4a–4b**: Figure selection, enhancement, and new figure generation. WebSearch retrieves current visualization best practices for the figure type before code is written.
2. **4c**: Multi-panel assembly with unified styling, panel labels, and figure legends.
3. **4d**: AI visual inspection — Bio-Framework reads each PNG preview via the Read tool, checks scientific content, formatting, and cross-figure consistency.
4. **4f**: User Visual Review gate — figures are displayed to you one by one with a one-sentence finding summary. You approve or request adjustments. Up to 2 revision rounds. Workflow does not advance until you confirm.

All figures are generated at 300 DPI, with English-only labels enforced automatically.

### Scenario 3: SCI Manuscript Generation

Step 6 is triggered automatically after Step 5 (technical foundation) completes:

```
Step 6 workflow (21 independent sub-steps):
  6a.  Outline — narrative structure, figure-to-section mapping
  6b.  Core writing — Title, Abstract, Introduction, Methods,
       Results, Discussion, Figure Legends (one file per section)
  6c.  Refinement — language polishing and consistency check
       for all six sections
  6d.  Assembly — manuscript_draft.md + supplementary_materials.md
       + quality_report.md + reference resolution
  6e–6i. Submission Readiness Validation — data validation,
       claim review, structural completeness, cross-file sync,
       final submission_readiness.md
```

Manuscript compliance can be checked against any supported journal at any time:

```
/bio manuscript check --journal nature
/bio manuscript check --journal cell_reports
/bio manuscript check --journal jci
```

Each check validates word limits per section, required sections, figure format requirements, and reference style.

## A Trainable AI Research Assistant

Bio-Framework is not a one-time tool. It includes a complete personalization and knowledge accumulation system that grows with you — the longer you use it, the better it understands your analysis style and research needs.

### 1. Parameter Preference Memory

Your analysis preferences persist across all projects through the three-layer configuration system:

```
# Save your preferred clustering resolution globally
/bio config set clustering.resolution 0.6 --global

# All future projects automatically use 0.6
# Override to 1.2 for a specific project when needed
```

Preferences are stored in `~/.claude/bio-framework/rules/user_rules.yml`, covering 12 parameter domains including QC thresholds, visualization settings, differential expression standards, and more.

### 2. Automatic Experience Accumulation

When the AI encounters and solves a problem during analysis, the solution is automatically saved for future reuse:

```
Problem: Seurat FindMarkers() error "not enough cells"
Solution: Lower min.pct threshold to 0.1, check cluster 8 cell count
         ↓ Auto-saved to learned_solutions.yml
Next time this error occurs → existing solution applied directly
```

The experience store is located at `~/.claude/bio-framework/user_knowledge/learned_solutions.yml`, shared across projects, with solution details, applicability conditions, and reuse counts.

### 3. Custom Knowledge Extension

You can add your own knowledge files to the user layer. The framework **merges** them with built-in knowledge (union strategy) rather than replacing it:

- Add your lab's commonly used marker gene lists
- Add disease-specific analysis guidelines
- Add your own QC experience rules

These custom knowledge files are preserved during framework upgrades.

### 4. Analysis Context Persistence

Every project's analysis progress, key decisions, parameter choices, and findings are durably persisted:

| Persisted Content | File | Purpose |
|-------------------|------|---------|
| Workflow state | `workflow_state.yml` | Precise recovery after interruption |
| Phase summaries | `phase_summaries.yml` | Records key decisions and parameters per phase |
| Analysis findings | `findings.yml` | Tracks anomalies and insights |
| Context snapshot | `/bio prime` | Fast context restore in new sessions |

### The Cumulative Effect

**Project 1**: You tell the AI your QC standards, preferred resolution, and figure style.
**Project 3**: The AI already knows your preference for stricter QC thresholds, finer clustering, and specific color schemes.
**Project 10**: The AI has accumulated your domain experience, common problem solutions, and analysis habits — it has become a research assistant that understands your style.

## Quick Start

### Requirements

- Claude Code CLI (Claude subscription required)
- R >= 4.3.0 and/or Python >= 3.9.0 (for analysis execution)
- Operating system: macOS, Linux, or Windows (WSL2)

### Installation

```bash
# Step 1: Extract and install globally
tar -xzf bio-framework-v1.0.0.tar.gz
cd bio-framework-v1.0.0
./install.sh --global

# Step 2: Initialize your analysis project
cd /your/project/directory
bio-init
```

### Start Your First Analysis

```bash
# In your project directory, open Claude Code, then:
/bio help           # View available commands
/bio topic new      # Create and configure your TOPIC.yml
/bio start          # Launch the analysis workflow
```

Natural language commands are also supported:

```
"start analysis"          # equivalent to /bio start
"继续分析"                 # Chinese: resume analysis
"分析を続ける"             # Japanese: continue analysis
```

### Resume After Interruption

```bash
/bio continue       # Resume from exact checkpoint
/bio status         # View current workflow state and progress
```

### Check Manuscript Compliance

```bash
/bio manuscript check --journal nature
/bio manuscript check --journal cell_reports
/bio manuscript check              # generic check (all sections)
```
